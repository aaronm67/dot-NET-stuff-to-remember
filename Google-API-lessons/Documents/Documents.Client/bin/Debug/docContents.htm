Overview

Chamberlain_Web_myQ

This web project uses the following technologies:

  • WebForms
  • Knockout
  • jQuery
  • Ajax

Surprisingly, no form elements are used on this website. Instead, javascript is used to call methods decorated with the [WebMethod] attribute on the ASPX pages.

There are five base pages in the folder called “Supers”:

  • Super.aspx - the base page that all of the other pages inherit from and is used to manage the following things:
  • Manage user authentication
  • Switch between mobile and non mobile site options
  • Set the appropriate resource manager that is used for globalization
  • AccountSuper.aspx - a base page that has web methods for managing user profile data and changing passwords.
  • DeviceSuper.aspx - a base page that has a plethera of web methods for managing devices and setting up the Chamberlain internet gateway.
  • LoginSuper.aspx - a base page with web methods used to login users in and out of the site.
  • RegisterSuper.aspx - this page does not have any web methods.

All of the other ASPX pages on the site inherit from one of the aforementioned base pages.

Lastly, a class called ChamberlainCommunicationClient is used to facilitate communication between the middleware and the web methods.

Chamberlain_Services_MyQ_External

This solution uses the following technologies:

  • MVC Web API
  • Arrayent cloud service - it makes calls to this service and processes the responses

This straightforward Web API is the “middleware” layer used to communicate with databases and the Arrayent cloud service. I set up a new site in IIS on my local box using this codebase. This allowed me to step into the middleware and see what was going on. In many cases, HttpWebReqest  and HttpWebResponse classes are used to communicate with the Arrayent endpoints in the cloud.

Project Structure

The following solutions are related to the MyQ website in some fasion:

  • Chamberlain_Web_myQ - this is the VS2010, WebForms web solution.
  • Chamberlain_MyQ_Shared  - this is a VS 2012 solution that has a collection of “business dll” projects supposedly used by the website. However, it does not appear anything from these projects is actually used by the website at present. Instead, there is a file named DomainV3.cs in the Chamberlain_Web_myQ project that is used as a “shim” until the actual business DLLs are added to the bin folder.
  • SharedResources - this is a collection of dlls that are used to some extent in all of the other TFS projects
  • Chamberlain_Services_MyQ_External - this is a VS 2012 solution with an MVC Web API project used as “middleware” between Chamberlain_Web_myQ and the following endpoints:
  • Three Entity Framework data contexts
  • The Arrayent cloud service

It appears that this project does use the business dlls from the Chamberlain_MyQ_Shared solution
